## Public framing

The Navigation System and Dashboard are live as a first community draft. The right public framing is:

- They are shared for review, feedback, and progressive correction.
- They are not final governance decisions.
- They are not a claim that every measure is automated or production-ready.
- The Dashboard tracks measurable success signals, not internal task completion.
- The Navigation System explains direction, dependencies, cards, and work status.

Use this stage label when asked about maturity:

```text
Version 0.0.01 | Community Draft
```

The same Community Draft framing is also used on:

- The Playing Field
- Stables circular economy
- Our Banking System
- Ambassador Program

## Relationship between the two pages

The **Navigation System** answers:

- What are we building?
- Which stream, group, card, and app phase does an item belong to?
- What is already done, what is blocked, and what depends on what?
- Which Dashboard success measures are relevant to each card?

The **Dashboard** answers:

- What external or community success signals can be measured?
- Which indicators already have a baseline?
- Which indicators require manual collection, API credentials, or future automation?
- Which Navigation cards explain the work behind each measure?

The two pages now link to each other:

- Dashboard cards link back to related Navigation cards.
- Navigation cards link to related Dashboard measures.

## No fake data rule

The Dashboard must not show fake merchant, user, social, or directory numbers. Demo app profiles and synthetic app data are excluded from success metrics unless explicitly labeled as demo-only. If a measure is not available, StablesAgent should say it is **TBD**, **awaiting baseline**, or **needs API**, not invent a number.
